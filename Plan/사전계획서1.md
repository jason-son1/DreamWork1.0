# DreamWork 서버 개발 준비 및 사전 기획서

본 문서는 플러그인을 설치하고 설정을 건드리기(Implementation) 전, **반드시 미리 준비하고 결정해야 할 사항(Pre-production)**을 정리합니다. 이 준비 과정을 거쳐야 설정 시간을 획기적으로 단축할 수 있습니다.

## 1. 사전 데이터 기획 (Data Planning)

**가장 중요합니다.** 플러그인 설정 파일(`config.yml`)을 열었을 때 고민하지 않고 바로 값을 입력할 수 있도록, 아래 내용을 **엑셀(Spreadsheet)**로 미리 작성해야 합니다.

### A. 직업 밸런스 데이터 (Jobs & RPG)

- **준비물:** `직업_경험치_테이블.xlsx`
    
- **포함해야 할 내용:**
    
    1. **5개 직업 정의:** 광부, 농부, 어부, 탐험가, 사냥꾼 (영어 ID: Miner, Farmer, Fisher, Adventurer, Hunter)
        
    2. **행동별 보상표:**
        
        - (예: 광부) 돌 캐기 = 0.5원 / 1xp, 다이아 캐기 = 50원 / 100xp
            
        - (예: 농부) 밀 수확 = 1원 / 2xp
            
    3. **레벨업 곡선:** 1레벨부터 100레벨까지 필요한 경험치 공식 (예: `이전레벨 * 1.2`)
        
    4. **스탯 매핑:** 각 직업이 레벨업 시 어떤 AuraSkills 스탯을 얼마나 줄지 수치 확정.
        

### B. 경제 및 상점 데이터 (Economy)

- **준비물:** `상점_가격표.xlsx`
    
- **포함해야 할 내용:**
    
    1. **서버 물가 기준:** 빵 1개 가격을 얼마로 할 것인가? (기준점, 예: 100원)
        
    2. **매입/매도 리스트:** 야생에서 얻는 모든 아이템의 상점 판매가.
        
    3. **건축 자재 가격:** 타운 건축에 필요한 블록 구매가 (회수 불가능하게 비싸게 설정).
        

### C. 등급 및 권한 계층 (Permissions)

- **준비물:** `권한_계층도.txt`
    
- **결정 사항:**
    
    1. **그룹 구분:** `Guest` (뉴비) -> `Citizen` (주민) -> `Noble` (후원자/고인물) -> `Admin`
        
    2. **그룹별 차등 권한:**
        
        - 뉴비: 텔레포트 딜레이 5초, 홈 셋 1개.
            
        - 주민: 텔레포트 딜레이 3초, 홈 셋 3개, 타운 생성 가능.
            

## 2. 필수 플러그인 및 라이브러리 확보 (Software Acquisition)

서버 구동을 위해 다운로드해 두어야 할 파일 목록입니다. **버전 호환성(1.20.x ~ 1.21.x)**을 반드시 맞춰야 합니다.

|            |                                                 |                                  |
| ---------- | ----------------------------------------------- | -------------------------------- |
| **구분**     | **준비해야 할 플러그인 (.jar)**                          | **비고 (필수 의존성)**                  |
| **코어**     | **PaperMC** (서버 구동기)                            | 일반 Spigot보다 최적화됨.                |
| **기반**     | **Vault**, **PlaceholderAPI**, **LuckPerms**    | 모든 플러그인의 연결고리.                   |
| **월드**     | **Multiverse-Core**, **Multiverse-Inventories** | 월드별 인벤토리 분리 필수.                  |
| **경제/기능**  | **EssentialsX** (Core, Chat, Spawn)             | 기본적인 명령어 및 경제 시스템.               |
| **직업/RPG** | **Jobs Reborn**, **AuraSkills**                 | 핵심 콘텐츠.                          |
| **타운**     | **Towny Advanced**, **SiegeWar** (선택)           | Towny가 가장 안정적임.                  |
| **건축**     | **FastAsyncWorldEdit (FAWE)**                   | WorldEdit의 최적화 버전 (서버 안 터짐).     |
| **몹/템**    | **MythicMobs**, **MythicCrucible**              | 커스텀 몹과 아이템 제작.                   |
| **UI/UX**  | **DeluxeMenus**, **DecentHolograms**            | 메뉴 및 홀로그램.                       |
| **리소스**    | **ItemsAdder** 또는 **Oraxen** (선택)               | 리소스팩 없이 하려면 **HeadDatabase** 추천. |

## 3. 리소스 및 에셋 준비 (Asset Preparation)

자체 텍스처를 만들지 않더라도, 서버를 꾸미기 위한 최소한의 준비가 필요합니다.

### A. 시각 자료 (Visuals)

- **유니코드 아이콘 모음:** 메뉴(GUI)나 채팅에 넣을 특수문자 준비 (예: ⚔, ⛏, ⛃).
    
- **컬러 코드표:** 서버의 테마 컬러 결정 (예: 메인색 `&6`(Gold), 강조색 `&b`(Aqua)).
    
- **Head Database 검색:**
    
    - `minecraft-heads.com` 사이트에서 햄버거, 모니터, CCTV, 우체통 등 장식용 머리의 "Value 값"을 미리 찾아 메모장에 저장. (건축 및 메뉴 아이콘용)
        

### B. 건축 청사진 (Schematics)

- **기획서의 '청사진 시스템'을 위해 필요한 것:**
    
    - 유저에게 제공할 **기본 건축물 샘플** (작은 집, 가로등 등)을 크리에이티브 월드에서 미리 짓고 `.schem` 파일로 저장해두어야 함.
        
    - 테스트용 건물을 하나 지어서 `//copy` -> `//schem save test_house` 해보는 과정 필요.
        

## 4. 서버 폴더 구조 설계 (Directory Structure)

무작정 `plugins` 폴더에 다 넣는 것이 아니라, 관리하기 쉽게 구조를 머릿속으로 그려야 합니다.

### A. 월드 구조 (World Management)

`Multiverse`로 생성할 월드 이름을 미리 정합니다.

1. `world` (로비/스폰): 파괴 불가능, 튜토리얼 및 포탈 존재.
    
2. `survival` (야생): 자원 채취용, 주기적 초기화 가능성 있음.
    
3. `town` (타운): Towny 플러그인 적용, 영구 보존, 평지 혹은 넓은 지형.
    
4. `design` (설계): PlotSquared 적용, 크리에이티브 모드, 청사진 제작용.
    

### B. 파일 백업 계획

- 플러그인 설정이 꼬였을 때를 대비해, **'순정 상태의 Config 폴더'**와 **'수정 중인 폴더'**를 분리해서 관리할 준비.
    

## 5. 단계별 적용 로드맵 (Implementation Roadmap)

**이 순서대로 진행해야 오류(Error)를 잡기 쉽습니다.**

1. **인프라 단계:**
    
    - 서버 켜기 -> `LuckPerms`로 관리자 권한 얻기 -> `Vault/Essentials`로 돈 기능 확인.
        
    - `Multiverse`로 월드 4개(로비, 야생, 타운, 설계) 생성 및 이동 확인.
        
2. **RPG 이식 단계 (가장 시간 많이 소요):**
    
    - `Jobs Reborn` Config 열기 -> 준비한 **엑셀 데이터** 옮겨 적기.
        
    - `AuraSkills` Config 열기 -> 직업별 스탯 가중치 입력.
        
3. **타운 및 건축 단계:**
    
    - `Towny` 설정에서 유지비, 마을 생성 비용 입력.
        
    - `FAWE` 권한 설정 (설계 월드에서만 `//copy` 가능하게 LuckPerms 설정).
        
4. **연동 및 포장 단계:**
    
    - `DeluxeMenus` 설치 -> `/메뉴` 명령어로 직업, 상점, 워프를 여는 GUI 제작.
        
    - `PlaceholderAPI` 명령어 입력 (`/papi ecloud download ...`) 하여 변수 활성화.
