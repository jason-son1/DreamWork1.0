### 2. 스킬 상세 명세 (Skill Mechanics)

액티브 스킬은 '미끼/떡밥' 아이템 사용으로, 패시브는 낚시 효율 증가로 구현합니다.

#### 2.1. [패시브] 입질 감지 (Sensitivity)

- **기능:** 낚시찌가 물고기를 무는 대기 시간(Wait Time) 단축 및 내구도 보존.
    
- **구현 로직:** `PlayerFishEvent` (CAUGHT_FISH) 발생 시점이 아니라, 낚싯대를 던질 때(`FISHING`) `Lure` 인챈트 레벨을 내부적으로 높게 계산하거나 NMS로 `hook`의 `wait time`을 조작.
    

|**레벨 (Lv)**|**입질 시간 단축**|**추가 효과**|
|---|---|---|
|**10 ~ 29**|5% 단축|없음7.|
|**30 ~ 49**|15% 단축|쓰레기 낚을 확률 대폭 감소8.|
|**50 ~ Max**|30% 단축|낚시 중 내구도 소모 무시 확률 20%9.|

#### 2.2. [패시브] 어종 지식 (Fish Knowledge)

- **기능:** 물가에 서거나 낚싯대를 들었을 때 해당 바이옴 정보 출력.
    
- **구현 로직:** `PlayerMoveEvent` (물가 접근 시) 또는 낚싯대 들고 `Shift + 우클릭`.
    

|**레벨 (Lv)**|**표시 정보 (Output)**|
|---|---|
|**기본**|현재 바이옴 이름 ("따뜻한 바다").|
|**20+**|주요 서식 어종 ("대구, 복어 서식지")10.|
|**50+**|**희귀/대형 어종** 출현 확률 및 현재 수역의 수심 정보11.|

#### 2.3. [아이템 액티브] 떡밥 투척 (Chumming)

- **아이템:** 어부 전용 **떡밥(Chum)**.
    
- **사용법:** 물을 보고 `우클릭`.
    
- **효과:** 해당 위치에 파티클(Fish School) 생성. 해당 위치에 찌를 던지면 효과 발동12.
    

|**떡밥 종류**|**지속 시간/횟수**|**효과**|
|---|---|---|
|**지렁이 (Worm)**|1회|입질 속도 즉시 단축 (Fast Bite)13.|
|**반짝이는 루어**|3회|보물(인챈트북 등) 획득 확률 증가14.|
|**크릴 새우 (Krill)**|1회|**Tier 3 대형 어종(참치, 대왕연어)** 낚을 확률 부여15.|

### 3. 아이템 및 데이터 구조 (Items & PDC)

바닐라 물고기 아이템에 PDC 데이터를 입혀 **일반 물고기**와 **살아있는/특수 물고기**를 구분합니다.

#### 3.1. 낚싯대 진화 (Rod Evolution)

- **기본 구조:** `Material.FISHING_ROD` + PDC Data.
    

|**등급**|**아이템 명**|**제작/해금 조건**|**PDC/Lore 특징**|**특수 능력**|
|---|---|---|---|---|
|**기본**|**낡은 낚싯대**|기본 지급|`Tier: 0`|바닐라와 동일16.|
|**숙련**|**숙련가의 낚싯대**|Lv.20 + 재료|`Tier: 1`|**월척 확률 증가:** 크기(Size) 롤링 시 가중치 부여17.|
|**전설**|**강태공의 낚싯대**|Lv.50 + 보물|`Tier: 2`|**미끼 절약:** 20% 확률로 미끼 소모 안 함18.|

#### 3.2. 특수 아이템 및 물고기 데이터

1. **살아있는 물고기 (Live Fish)**
    
    - **Item:** 물고기 아이템 or 물 양동이.
        
    - **PDC:** `status: live`, `size: 85.4`, `species: tuna`.
        
    - **획득:** **살림망(Fish Keep)** 아이템을 인벤토리에 소지한 채 낚시 성공 시 자동 저장19.
        
    - **가치:** 죽은 물고기 대비 2~3배 가격20.
        
2. **계측자 (Ruler)**
    
    - **Item:** `Material.PAPER` (Custom Model).
        
    - **기능:** 인벤토리에 있으면 낚시 성공 시 채팅창에 "[75cm 대왕 연어]를 낚았습니다!" 메시지 출력 및 아이템 Lore에 기록21.
        
3. **대형 어종 (Tier 3)**
    
    - **참치 (Tuna):** `Material.COD` + Custom Name/Lore.
        
    - **대왕 연어 (King Salmon):** `Material.SALMON` + Custom Name/Lore.
        
    - _Note:_ 오직 **크릴 새우** 떡밥 사용 시에만 등장 테이블에 포함됨22.

### 5. 경제 및 타운 시스템 연동 (Economy & Town)

어부는 **'신선도(Freshness)'**를 기준으로 납품처가 나뉩니다.

#### 5.1. 가공 및 요리 (Processing & Sashimi)

- **생선 손질 (Filleting):**
    
    - 타운 내 `가공 NPC` 또는 어부 자신이 칼(Sword)로 생선과 조합.
        
    - 결과물: **회(Sashimi)** + 부산물(뼈, 비늘).
        
    - **회(Sashimi):** 즉시 섭취 가능한 포션형 식량(체력 회복, 수중 호흡 등 버프)26.
        
- **부산물 활용:**
    
    - `뼈다귀`: 농부의 비료 재료로 판매.
        
    - `비늘/어유`: 연금술 또는 갑옷 제작 재료로 판매27.
        

#### 5.2. 경제 순환 모델 (The Loop)

1. **어부**는 미끼를 써서 물고기를 낚음.
    
2. **살아있는 물고기(Live)**는 타운의 **아쿠아리움**에 비싸게 납품 (관상용/수집용)28.
    
3. **죽은 물고기**는 **회**로 가공하여 **사냥꾼(Hunter)**이나 **탐험가(Adventurer)**에게 전투 식량으로 판매29.
    
4. 획득한 돈으로 더 좋은 **미끼(크릴)**와 **낚싯대**를 구매하여 희귀 어종 공략.

### 💡 개발자(User)를 위한 구현 팁 (Technical Note)

1. **Event Listener:**
    
    - `PlayerFishEvent`:
        
        - `state == CAUGHT_FISH` 확인.
            
        - `event.getCaught()`로 엔티티 확인 후, 확률 테이블(Config)에 따라 `ItemEntity`를 **교체**.
            
        - _Logic:_ 바닐라 물고기 드롭을 `event.setCancelled(true)` 하지 말고, `itemEntity.setItemStack(customFish)`로 바꿔치기하는 것이 호환성에 좋음30.
            
        - 미끼(Off-hand) 체크 로직 추가.
            
2. **Config 구성 (`jobs/fisher.yml`):**
    
    YAML
    
    ```
    skills:
      sensitivity:
        reduce-tick: [0, 5, 15, 30] # 입질 단축 %
    
    items:
      bait_krill:
        allowed-fish: [TUNA, KING_SALMON]
        multiplier: 1.5
    
    fish-table:
      TUNA:
        base-material: COD
        display-name: "&b참다랑어"
        chance: 5.0 # 크릴 사용 시 5%
        min-size: 50.0
        max-size: 150.0
    ```
    
    - 물고기 종류별 확률, 크기 범위, 필요 미끼를 YAML로 정의합니다 31.
        
3. **PDC & NBT:**
    
    - 물고기 생성 시:
        
        Java
        
        ```
        // Pseudo Code
        meta.getPersistentDataContainer().set(keySpecies, STRING, "TUNA");
        meta.getPersistentDataContainer().set(keySize, DOUBLE, 120.5);
        meta.getPersistentDataContainer().set(keyLive, BYTE, 1); // 1=Live, 0=Dead
        ```
        
    - 상점 플러그인에서 `keyLive`가 1인 경우 가격을 3배로 책정하도록 연동 (`DreamShop` 구현 시).
