# [개정안] 어부 (Fisher): 심해의 탐구자 및 항해사

## 1. 기본 설정 및 진행 방향 (Core Concept)

### 1.1. 기본 컨셉

- **초반 (Early Game):** 기존 마인크래프트의 낚시 메커니즘을 그대로 유지합니다. 일반 낚싯대와 바닐라 미끼(없는 상태)로 시작하며, 기본적인 물고기(대구, 연어)를 낚습니다.
    
- **중/후반 (Mid/Late Game):** 어부 전용 레벨과 미션(예: 연어 100마리 낚기, 특정 바이옴 낚시 성공 등)을 달성함에 따라 '전문 낚시꾼'으로 성장합니다.
    
- **핵심 변화:** 단순히 기다리는 낚시에서 벗어나, **'미끼(Bait)'**를 사용하여 원하는 어종을 노리고, **'낚싯대 개조'**를 통해 입질 시간을 줄이거나 보물 획득률을 높이는 **장비 성장형 구조**로 전환합니다.
    
- **리소스 구현:** 별도의 텍스처 모드 없이도 자체 플러그인(DreamWork)의 PDC/NBT 시스템을 활용하여, 기존 낚싯대와 물고기 아이템에 특수 데이터를 부여해 구분합니다.
    

### 1.2. 경험치 및 보상 테이블 (Config 예시)

모든 수치는 설정 파일(YAML)에서 운영자가 수정 가능합니다.

|**대상 (Target)**|**등급 (Tier)**|**기본 경험치**|**기본 수입 (D)**|**비고**|
|---|---|---|---|---|
|대구, 연어|Tier 1|1.0|0.5|모든 수역에서 등장 (기본)|
|복어, 열대어|Tier 2|3.0|2.0|특정 바이옴 또는 미끼 필요|
|**대형 어종** (참치, 돔 등)|Tier 3|10.0|15.0|전용 미끼 사용 시 등장 (NBT로 구현된 바닐라 물고기 변형)|
|**심해의 보물**|Special|50.0|100.0|인챈트북, 귀중품 등|
|쓰레기 (가죽장화 등)|Waste|0.1|0|레벨이 오를수록 낚을 확률 감소|

## 2. 스킬 (Skills) - 점진적 해금 시스템

쿨타임이 있는 액티브 스킬(그물 던지기 등)을 삭제하고, **레벨과 미션 진행도**에 따라 자동으로 적용되는 **패시브**나 **소모품 사용 권한**으로 변경합니다.

### 2.1. [패시브] 입질 감지 (Sensitivity)

- **기능:** 낚시찌가 물고기를 무는 대기 시간(Wait Time)을 단축시킵니다.
    
- **단계별 해금 (Configurable):**
    
    - **Lv. 1 ~ 9:** 효과 없음 (바닐라 상태).
        
    - **Lv. 10 + [미션: 대구 50마리]:** 입질 시간 5% 단축.
        
    - **Lv. 30 + [미션: 다양한 어종 5종 낚기]:** 입질 시간 15% 단축 + 쓰레기 낚을 확률 대폭 감소.
        
    - **Lv. 50 (Master):** 입질 시간 30% 단축 + 낚시 중 내구도 소모 무시 확률 20%.
        

### 2.2. [패시브] 어종 지식 (Fish Knowledge)

- **기능:** 특정 바이옴이나 물가에 섰을 때, 해당 지역에서 낚을 수 있는 주요 어종 정보를 채팅창에 띄워줍니다. (광부의 광맥 탐지와 유사)
    
- **효과:** 레벨이 오를수록 물고기의 정확한 크기(cm)나 희귀 어종의 존재 유무를 파악할 확률이 늘어납니다.
    

### 2.3. [아이템 액티브] 떡밥 투척 (Chumming)

- **변경점:** 기존의 '그물 던지기' 스킬 삭제 → **'떡밥(Chum)' 아이템** 사용으로 변경.
    
- **기능:** 어부가 제작하거나 구매한 '떡밥'을 물에 던지면(우클릭), 해당 위치에 **일시적인 어군(Fish School)** 파티클이 생성됩니다.
    
- **효과:**
    
    - 떡밥이 뿌려진 곳에 찌를 던지면 입질이 즉시 오거나(Fast Bite), 평소에 잡히지 않는 **'히든 물고기'**가 잡힐 확률이 생깁니다.
        
    - 어부 레벨이 높아야 상위 등급의 떡밥을 효과적으로 사용할 수 있습니다.

## 3. 아이템 (Items)

**Oraxen/ItemsAdder**가 있다면 좋지만, 없다면 **바닐라 아이템 + NBT/Lore** 시스템을 적극 활용합니다.

### 3.1. 미끼 시스템 (Bait System)

농부의 '비료'와 대응되는 시스템입니다. 낚싯대의 보조 슬롯(혹은 인벤토리 내 특정 위치)에 두고 낚시를 하면 소모됩니다.

- **지렁이 (Worm):** 가장 기초적인 미끼. 입질 속도 소폭 증가.
    
- **반짝이는 루어 (Shiny Lure):** 보물(인챈트북, 안장 등)을 낚을 확률 증가.
    
- **크릴 새우 (Krill):** 특정 고급 어종(참치, 연어 대뱃살 등)을 낚기 위해 필수.
    
- **심해의 발광체:** 심해/동굴 낚시 전용. 아귀나 발광 오징어 포획률 증가.
    

### 3.2. 장비: 낚싯대의 진화 (Rod Evolution)

광부의 곡괭이처럼 낚싯대도 성장합니다.

- **낡은 낚싯대 (Old Rod):** 바닐라 기본 낚싯대.
    
- **숙련가의 낚싯대 (Pro Rod):**
    
    - _제작 조건:_ 어부 Lv.20 + 특정 재료.
        
    - _효과:_ 내구도 증가 + "월척(크기가 큰 물고기)"을 낚을 확률 증가.
        
- **전설의 강태공 낚싯대 (Master's Rod):**
    
    - _제작 조건:_ 어부 Lv.50 + 심해의 보물 재료.
        
    - _효과:_ 미끼 소모 확률 20% 감소 + 당기는 속도 증가 + **'전설의 물고기'** 입질 가능.
        
    - _강화:_ 대장간이나 어부 전용 NPC를 통해 낚싯대에 특수 능력(내구도 회복, 행운 등)을 부여.
        

### 3.3. 기타 아이템

- **계측자 (Ruler):** 인벤토리에 소지 시, 낚은 물고기의 길이를 자동으로 측정하여 Lore에 기록 (예: `[75cm 대왕 연어]`).
    
- **살림망 (Fish Keep):** 낚은 물고기를 바로 아이템화하지 않고, 살아있는 상태(Live)로 보관하는 가방 역할. (아쿠아리움 납품용)

## 4. 경제 및 타운 (Economy & Town)

### 4.1. 생선 손질 및 요리 (Filleting & Cuisine)

단순히 생선을 화로에 굽는 것을 넘어, **'회(Sashimi)'**와 **'가공'**을 통해 부가가치를 창출합니다.

- **가공 NPC (Fishmonger):** 타운에 고정 배치된 NPC.
    
- **기능:**
    
    1. **회 뜨기:** 잡은 물고기를 가져가면 수수료를 받고 '회'로 만들어줍니다. (어부 레벨이 높으면 직접 도구로 가능하게 설정 가능)
        
        - _회 효과:_ 섭취 시 즉시 체력 회복, 수중 호흡 등 전투/탐험 버프 제공. (상점 판매가 높음)
            
    2. **부산물 추출:** 낚시 중 얻은 쓰레기나 잡어를 분해하여 '뼈다귀(비료 재료)', '비늘(갑옷 재료)', '어유(포션 재료)' 획득.
        

### 4.2. 아쿠아리움 납품 (Aquarium Delivery)

- 타운 내 **'수족관 관리자 NPC'**에게 **살아있는 물고기(Live Fish)**를 납품합니다.
    
- 죽은 물고기(아이템)보다 살아있는 상태(물양동이 담긴 상태 혹은 살림망 NBT)가 2~3배 비쌉니다.
    
- **도감 완성:** 서버 내의 모든 종류/크기의 물고기를 납품하면 명예 보상(칭호, 설치형 트로피)을 받습니다.
    

### 4.3. 어부의 상점

- 어부가 만든 **'특제 회'**나 **'심해 전리품(산호, 프리즈머린 등)'**은 건축가와 사냥꾼 유저들에게 직접 판매하여 수익을 올립니다.
    
- 일반적인 물고기는 타운 상점에 납품하여 기초 자금을 마련합니다.

## 5. 이벤트 (Event)

**돌발 이벤트 삭제:**

- 크라켄 출현, 폭풍우, 황금 조류 등의 랜덤 이벤트는 피로도를 유발하므로 **삭제**합니다.
    
- 대신, 잔잔하게 낚시에 집중하며 '월척'을 낚았을 때 서버 전체에 메시지가 뜨는 정도의 소소한 성취감에 집중합니다. (예: _"플레이어 000님이 전설적인 120cm 참다랑어를 낚았습니다!"_)

## 6. 개발자 참고사항 (Implementation Note)

- **PDC/NBT 데이터:**
    
    - 낚인 물고기에 `fish_type: tuna`, `size: 85.4`, `freshness: live` 같은 태그를 부여하여, 일반 바닐라 물고기와 시스템적으로 완전히 분리해야 합니다.
        
    - 예: 바닐라 '익히지 않은 연어' 아이템을 사용하되, 이름은 '대왕 연어', Lore에는 '크기: 100cm'가 적혀 있음.
        
- **미끼 구현:**
    
    - 낚시 이벤트(`PlayerFishEvent`) 발생 시, 보조손(Off-hand)이나 인벤토리의 특정 아이템(미끼)을 감지하여 `CAUGHT_FISH` 확률 테이블을 변경하는 로직이 필요합니다.
        
- **Config 구조 (jobs/fisher.yml):**
    
    YAML
    
    ```
    level-up-curve: "level * 100 + 50"
    skills:
      sensitivity: # 입질 감지
        unlock-level: 10
        reduce-wait-time: 0.05 # 5% 감소
    items:
      bait_worm:
        catch-rate-multiplier: 1.2
        allowed-fish: [COD, SALMON, PUFFERFISH]
      bait_krill:
        catch-rate-multiplier: 1.5
        allowed-fish: [TUNA, KING_SALMON] # Custom Fish IDs
    fishing-drop-table:
      # 바이옴, 미끼, 낚싯대 등급에 따른 드롭 확률 설정
    ```
